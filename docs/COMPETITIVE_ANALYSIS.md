## 竞品对比

### Scrapy
- **定位**：大型爬虫框架，适合大规模爬取
- **优势**：异步引擎、中间件体系、管道/管道插件、内置去重、Feed 导出
- **劣势**：学习曲线陡峭、配置复杂、不适合一次性快速抓取

### Crawlee (Playwright/Puppeteer)
- **定位**：现代化爬虫框架，浏览器自动化
- **优势**：JS 渲染、自动重试/代理轮换、请求队列、存储适配器
- **劣势**：Node.js 生态为主、Python 支持较弱

### HTTPie / xh
- **定位**：HTTP 客户端，不是爬虫
- **优势**：极简、人类友好
- **劣势**：无爬取能力

### curlie
- **定位**：curl 的交互式包装器
- **优势**：Tab 自动补全、表单自动填充
- **劣势**：不是爬虫

### httpx (Python library)
- **定位**：HTTP 客户端库
- **优势**：支持 HTTP/2、异步
- **劣势**：库而非 CLI，需要自己写逻辑

## wscraper 的差异化机会

**wscraper 的定位应该是：开箱即用的命令行爬虫，一行命令完成网页数据提取。**

### 差异化优势
1. **极简 CLI** — 不需要写 Python 代码就能用
2. **中文友好** — 文档和错误信息中文
3. **轻量级** — 依赖少，安装快
4. **实用主义** — 内置常用场景（表格、链接、定时）

## 功能路线图

### v0.3 — 核心增强（建议优先）

1. **异步抓取** — 用 `aiohttp`/`httpx` 替换 `requests`
   - 大幅提升多 URL 并发性能
   - 为翻页/批量场景提速 5-10x

2. **Sitemap 解析** — `--sitemap` 参数
   - 自动从 `sitemap.xml` 发现所有可抓取 URL
   - 支持 sitemap index 递归展开

3. **递归爬取** — `--depth N` 参数
   - 从入口页自动发现并抓取子页面
   - 可设置最大深度和同域限制

4. **结果过滤/管道** — `--filter` 参数
   - 支持简单的 JSONPath 过滤
   - 支持 `--unique` 去重

### v0.4 — 高级功能

5. **JS 渲染** — 集成 `playwright`（可选依赖）
   - `--render` 参数启用浏览器渲染
   - 支持 SPA/动态页面

6. **高级反检测**
   - 指纹轮换（TLS fingerprint, canvas 等）
   - 请求头指纹模拟
   - 行为模拟（随机滚动、点击间隔）

7. **变更检测** — `--watch` 参数
   - 定时抓取并比较上次结果
   - 有新内容时通知（邮件/Slack webhook）

8. **数据导出增强**
   - Markdown 表格输出
   - SQLite 数据库导出
   - API 推送（支持 Webhook）

### v0.5 — 生态完善

9. **插件系统**
   - 支持自定义提取器脚本
   - 支持后处理管道

10. **配置化任务**
    - YAML/JSON 配置文件
    - 预定义任务模板

11. **GitHub Actions / CI 集成**
    - 定期运行爬虫任务
    - 自动 PR 更新数据

## 短期建议

> 最快见效的三个改动：

1. **添加 `--help` 中文示例** — 降低上手门槛
2. **添加 `--sitemap`** — 一键发现全站 URL
3. **添加 `--depth`** — 基础递归爬取

这三个改动工作量不大，但显著提升工具实用性。
